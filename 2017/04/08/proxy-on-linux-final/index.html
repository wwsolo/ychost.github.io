linux 翻墙终极版

2017/04/08 系统 推荐 Linux Shadowsocks Privoxy
　 本文为「原创」内容，如需转载请注明出处！ 　　　　　　　　 　　　

本教程使用了shadowskcs和privoxy，可以让linux的终端和浏览器同时翻墙

总览

  1. 注册一枝红杏账号
  2. 安装Shadowsocks-qt5并配置
  3. 安装Privoxy并配置
  4. 测试效果

购买一枝红杏的SS套餐

一枝红杏是的一家shadowsocks服务提供商，一年的价格为99元，经测试速度还不错在youtube上面可以流畅观看1080P的视频，当然你也可以选择其它的shadowsocks服务提供商

  1. 点开一枝红杏并点击订购服务
  2. 点击标准型套餐并订购
  3. 添加到购物车，输入密码，此密码为shadowsck客户端连接需要用到的
  4. 填入页面下的促销代码，然后点击验证，最后点击结账
  5. 弹出注册界面，填好相关信息并注册
  6. 点击账单界面的支付宝付款，然后进行付款
  7. 进入我的SS账号->管理账号->查看详情就可以看见刚刚购买SS套餐详情了

安装Shadowsocks-qt5

安装过程请参考Shadowsocks-qt5的官方wiki

sudo add-apt-repository ppa:hzwhuang/ss-qt5
sudo apt-get update
sudo apt-get install shadowsocks-qt5

装好后的效果

配置Shadowsocks-qt5

  1. 打开刚刚购买的一枝红杏SS套餐详情界面，并点击新加坡节点（新加坡的速度相对比较快）的二维码按钮，然后就会弹出SS配置二维码

  2. 打开Shadowsocks-qt5软件，然后点击链接->添加->扫描屏幕上的二维码，出现配置界面然后点击OK，然后点击连接

  3. 打开网络设置并设置代理我这里是deepin的发行版，不同发行版的操作界面可能不大一样，注意这里用的是自动代理指向的是一个pac文件用于实现局部翻墙，我这里已经生成了一个pac文件有需要的自行下载，当然也可以自己用工具根据gfwlist.txt生成最新的pac文件

  4. 点击应用到整个系统，然后测试是否能打开Google

  5. 此时浏览器已经可以翻墙了，但是终端还是不行，因为我们走的时候Socks5的代理，然而终端要走http的代理，所以需要使用Privoxy来实现对其桥接

安装配置Privoxy

sudo apt-get install privoxy

  1. 打开配置文件 /etc/privoxy/config

     sudo vim /etc/privoxy/config

    找到forward-socks5（如果是forward-socks5t改成forward-socks5） 并将后面的值改成 127.0.0.1:1080，将http代理到socks5

  2. 重启privoxy 
     sudo service privoxy restart
  3. 配置终端的代理，8118端口是默认端口，可以在config里面修改，测试wget去访问google发现终端已经翻墙了 
     export http_proxy=http://127.0.0.1:8118/
 export https_proxy=http://127.0.0.1:8118/
 wget www.google.com

  4. 这只是临时修改了代理的环境变量，重启终端会丢失的，所以可以将这两个环境变量配置在 ~/.bashrc 里面这样就始终有效了，但是由于是全局翻墙的所以这样会导致访问国内的服务很慢，所以还是设置临时翻墙比较好，当然 privoxy也是支持类似于pac那种局部翻墙的，但是语法上稍有区别，可以参考Privoxy 简明教程

总结

　　 Linux下面的全局翻墙设置很麻烦，不像windows或者mac在shadowsocks里面勾选全局翻墙就行了，一般来说浏览器翻墙就行了，当遇到npm或者gem，gradle在终端编译不过的时候一般都是被墙了，所以需要里面privoxy来设置代理转发给socks5实现翻墙
